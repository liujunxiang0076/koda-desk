# 第一阶段功能验证与测试

## 1. 文档目的

本文档用于指导 Koda Desk 第一阶段 MVP 的测试与验收。重点验证用户视角的完整行为、隐藏状态下的动画与资源占用、资源异常时的兜底能力，确保 MVP 在进入第二阶段前具备基础稳定性。

本阶段测试统一归类为：

- MVP 功能验收。
- 性能 / 资源占用验证。
- 异常与容错验证。

建议在开发计划中将本阶段命名为：

- MVP 测试与验收。
- MVP 验收与稳定性验证。

## 2. 测试范围

### 2.1 覆盖范围

- 应用启动。
- 透明无边框窗口。
- 宠物置顶显示。
- Canvas 加载 spritesheet。
- `pet.json` 读取。
- idle 动画播放。
- 拖拽移动。
- 右键菜单。
- 系统托盘。
- 显示 / 隐藏宠物。
- 退出应用。
- 隐藏时动画暂停。
- 隐藏时 CPU 占用不持续升高。
- 资源缺失或损坏时的兜底。

### 2.2 暂不覆盖

- 多宠物切换。
- 自动状态识别。
- 多状态动画切换。
- 点击宠物反应。
- 宠物缩放。
- 位置记忆。
- 开机自启。
- 完整设置页。
- 跨平台完整兼容性。

## 3. 测试环境

### 3.1 基础环境

- 平台：Windows。
- 应用形态：Tauri 桌面应用。
- 渲染方式：Canvas。
- 宠物资源：`spritesheet.webp` + `pet.json`。

### 3.2 建议准备

- 一个可正常加载的默认宠物资源目录。
- 一个缺失 `pet.json` 的测试资源目录。
- 一个 JSON 格式错误的 `pet.json`。
- 一个缺失 `spritesheet.webp` 的测试资源目录。
- 一个损坏或非图片内容的 `spritesheet.webp`。
- Windows 任务管理器或其他 CPU / 内存观察工具。
- 开发控制台、日志输出或终端输出观察入口。

### 3.3 测试记录字段

每项测试建议记录：

- 测试日期。
- 测试人。
- 应用版本或提交 ID。
- 操作系统版本。
- 测试资源目录。
- 实际结果。
- 是否通过。
- 问题描述。
- 关联缺陷或修复记录。

## 4. MVP 功能验收

该部分属于端到端功能验收，验证用户视角的完整行为：启动、透明窗口、动画、拖拽、右键菜单、托盘显示隐藏、退出等。

### 4.1 应用启动

目标：

- 验证应用能正常启动，并展示宠物主窗口和系统托盘入口。

前置条件：

- 默认宠物资源存在。
- `pet.json` 格式正确。
- `spritesheet.webp` 可正常读取。

步骤：

1. 启动应用。
2. 观察桌面是否出现宠物。
3. 观察系统托盘是否出现应用图标。
4. 查看控制台或日志是否有启动错误。

预期结果：

- 应用启动成功。
- 桌面出现宠物。
- 系统托盘出现应用入口。
- 无阻塞性错误。

通过标准：

- 宠物窗口和托盘入口均可见。
- 应用没有崩溃、卡死或反复重启。

### 4.2 透明无边框窗口

目标：

- 验证主窗口只显示宠物，不显示普通窗口 UI。

步骤：

1. 启动应用。
2. 将宠物移动到不同颜色的桌面背景或其他窗口上方。
3. 观察宠物周围是否存在矩形背景。
4. 观察是否有标题栏、边框、系统按钮。

预期结果：

- 背景透明。
- 无标题栏。
- 无边框。
- 无最小化、最大化、关闭按钮。

通过标准：

- 只看到宠物本体，不看到窗口矩形背景。
- 透明区域不出现黑底、白底或系统默认背景。

### 4.3 宠物置顶显示

目标：

- 验证宠物默认置顶，同时不影响其他应用操作。

步骤：

1. 启动应用。
2. 打开一个普通窗口，例如浏览器、编辑器或资源管理器。
3. 将普通窗口移动到宠物所在区域。
4. 切换焦点到普通窗口。
5. 在普通窗口中输入或点击。

预期结果：

- 宠物仍显示在普通窗口上方。
- 普通窗口仍可正常获得焦点。
- 键盘输入和鼠标点击不被异常拦截。

通过标准：

- 宠物保持置顶。
- 其他应用可正常操作。

### 4.4 Canvas 加载 spritesheet

目标：

- 验证 Canvas 能正确加载并绘制默认 spritesheet。

步骤：

1. 启动应用。
2. 观察宠物是否正常显示。
3. 查看控制台或日志中的资源加载信息。
4. 检查是否出现空白、错位、拉伸或裁切异常。

预期结果：

- `spritesheet.webp` 加载成功。
- Canvas 能绘制正确帧。
- 宠物比例正确。

通过标准：

- 宠物图像清晰可见。
- 没有明显错位、空白帧或错误裁切。

### 4.5 读取 pet.json

目标：

- 验证应用按 `pet.json` 中的帧尺寸、网格和动画配置渲染 idle 动画。

步骤：

1. 确认默认资源目录包含 `pet.json`。
2. 启动应用。
3. 查看 idle 动画是否按配置循环。
4. 查看控制台或日志是否输出解析错误。

预期结果：

- `pet.json` 读取成功。
- `frame.width`、`frame.height`、`columns`、`rows` 被正确使用。
- `animations.idle` 被正确识别。

通过标准：

- idle 动画正常播放。
- 没有 manifest 解析错误。

### 4.6 idle 动画循环播放

目标：

- 验证宠物默认播放 idle 动画，并按 8-12 FPS 左右循环。

步骤：

1. 启动应用。
2. 连续观察宠物 30 秒。
3. 记录动画是否持续播放。
4. 观察是否出现停止、跳帧、错帧或闪烁。

预期结果：

- idle 动画持续循环。
- 动画速度自然。
- 没有明显卡顿或闪烁。

通过标准：

- 动画能稳定运行 30 秒以上。
- 体感帧率符合 8-12 FPS 的低帧率动画目标。

### 4.7 拖拽移动

目标：

- 验证用户可以拖拽宠物移动窗口。

步骤：

1. 启动应用。
2. 鼠标左键按住宠物区域。
3. 拖动到桌面另一个位置。
4. 松开鼠标。
5. 重复 3 次，包括快速拖动和慢速拖动。

预期结果：

- 宠物跟随鼠标移动。
- 松开后停留在目标位置。
- 拖拽过程不卡死。

通过标准：

- 拖拽行为稳定。
- 拖拽不会误触发右键菜单。
- 拖拽后动画继续播放。

### 4.8 右键菜单打开

目标：

- 验证用户可以通过右键打开宠物菜单。

步骤：

1. 启动应用。
2. 在宠物区域点击鼠标右键。
3. 观察菜单是否出现。
4. 检查菜单项是否包含隐藏、显示、退出。

预期结果：

- 右键菜单正常显示。
- 菜单项完整。
- 菜单不会被宠物窗口遮挡。

通过标准：

- 右键菜单能稳定打开。
- 菜单项文字和行为入口正确。

### 4.9 右键菜单隐藏宠物

目标：

- 验证右键菜单可以隐藏宠物窗口。

步骤：

1. 启动应用。
2. 右键宠物打开菜单。
3. 点击隐藏。
4. 观察宠物是否消失。
5. 观察托盘图标是否仍存在。

预期结果：

- 宠物窗口隐藏。
- 应用未退出。
- 托盘图标仍存在。

通过标准：

- 宠物不可见。
- 托盘仍能打开菜单。

### 4.10 托盘显示宠物

目标：

- 验证宠物隐藏后可以通过托盘恢复显示。

步骤：

1. 先通过右键菜单隐藏宠物。
2. 打开系统托盘菜单。
3. 点击显示宠物。
4. 观察宠物是否重新出现。

预期结果：

- 宠物重新显示。
- idle 动画恢复播放。
- 应用没有创建重复窗口。

通过标准：

- 隐藏后可以稳定恢复显示。
- 恢复后仍保持透明、无边框、置顶。

### 4.11 托盘隐藏宠物

目标：

- 验证托盘菜单可以隐藏宠物窗口。

步骤：

1. 启动应用并确认宠物可见。
2. 打开系统托盘菜单。
3. 点击隐藏宠物或显示 / 隐藏宠物。
4. 观察宠物是否消失。

预期结果：

- 宠物窗口隐藏。
- 托盘图标仍存在。

通过标准：

- 托盘隐藏功能稳定可用。

### 4.12 退出应用

目标：

- 验证右键菜单和托盘菜单都可以完整退出应用。

步骤：

1. 启动应用。
2. 通过右键菜单点击退出。
3. 观察宠物窗口和托盘图标是否消失。
4. 再次启动应用。
5. 通过托盘菜单点击退出。
6. 观察宠物窗口和托盘图标是否消失。
7. 检查任务管理器中是否残留应用进程。

预期结果：

- 宠物窗口关闭。
- 托盘图标消失。
- 应用进程退出。

通过标准：

- 两种退出入口都能完整结束应用。
- 无残留常驻进程。

## 5. 性能 / 资源占用验证

隐藏时动画暂停确认属于功能验证 + 性能验证。功能上验证隐藏宠物后动画不会继续跑；性能上验证不会在隐藏状态持续占 CPU。

### 5.1 idle 状态资源观察

目标：

- 建立 MVP 正常显示时的基础资源占用参考。

步骤：

1. 启动应用。
2. 等待 30 秒，让应用进入稳定状态。
3. 打开任务管理器。
4. 记录应用 CPU 占用。
5. 记录应用内存占用。
6. 连续观察 3 分钟。

预期结果：

- CPU 占用较低且稳定。
- 内存占用无持续增长趋势。
- 动画稳定播放。

通过标准：

- 空闲显示状态下没有持续高 CPU。
- 3 分钟内内存没有明显持续上涨。

记录项：

- 平均 CPU。
- 峰值 CPU。
- 初始内存。
- 3 分钟后内存。

### 5.2 隐藏时动画暂停

目标：

- 验证宠物隐藏后，Canvas 动画循环暂停或停止高频渲染。

步骤：

1. 启动应用。
2. 确认 idle 动画正在播放。
3. 通过右键菜单或托盘隐藏宠物。
4. 观察控制台、日志或内部调试输出中的动画状态。
5. 等待 1 分钟。
6. 通过托盘显示宠物。
7. 观察 idle 动画是否恢复。

预期结果：

- 隐藏后动画循环暂停。
- 显示后动画恢复。
- 不出现重复启动多个动画循环的问题。

通过标准：

- 隐藏期间没有继续按 8-12 FPS 绘制。
- 恢复显示后动画只运行一套循环。

建议实现辅助：

- 开发阶段可输出 `animation started`、`animation paused`、`animation resumed`。
- 可在调试模式记录当前 animation loop ID 或 tick 计数。

### 5.3 隐藏时 CPU 占用

目标：

- 验证宠物隐藏后不会持续占用 CPU。

步骤：

1. 启动应用。
2. 记录显示状态下 CPU 占用。
3. 隐藏宠物。
4. 等待 30 秒。
5. 连续观察 3 分钟 CPU 占用。
6. 重新显示宠物。
7. 再观察 CPU 是否恢复到正常显示状态。

预期结果：

- 隐藏后 CPU 占用下降或保持极低。
- CPU 不持续波动在明显高位。
- 重新显示后动画恢复，CPU 回到显示状态的合理范围。

通过标准：

- 隐藏状态下 CPU 不应因为动画循环持续运行而明显占用。
- 重新显示后没有 CPU 异常升高。

### 5.4 多次显示 / 隐藏稳定性

目标：

- 验证多次切换显示状态不会累积动画循环、窗口实例或资源占用。

步骤：

1. 启动应用。
2. 连续执行显示 / 隐藏 10 次。
3. 每次显示后观察动画是否正常。
4. 每次隐藏后观察宠物是否不可见。
5. 第 10 次显示后观察 CPU 和内存。

预期结果：

- 每次切换都成功。
- 没有重复窗口。
- 没有多个动画循环叠加。
- 资源占用没有明显累积。

通过标准：

- 10 次切换后应用仍稳定。
- CPU 和内存保持在合理范围。

## 6. 异常与容错验证

资源失败兜底测试属于异常场景 / 容错测试。它验证 `pet.json` 缺失、格式错误、`spritesheet.webp` 缺失时，应用不会卡死或崩溃，并能输出明确错误。

### 6.1 pet.json 缺失

目标：

- 验证缺少 `pet.json` 时应用不会崩溃。

步骤：

1. 准备一个没有 `pet.json` 的默认宠物资源目录。
2. 启动应用。
3. 观察应用是否卡死或崩溃。
4. 查看控制台或日志错误。
5. 尝试通过托盘退出应用。

预期结果：

- 应用不崩溃。
- 输出明确错误，例如 `pet.json not found`。
- 托盘入口可用。
- 用户可以退出应用。

通过标准：

- 应用保持可控。
- 错误信息能定位到 `pet.json` 缺失。

### 6.2 pet.json 格式错误

目标：

- 验证 `pet.json` 内容不是合法 JSON 时应用不会崩溃。

步骤：

1. 将 `pet.json` 修改为非法 JSON。
2. 启动应用。
3. 观察宠物窗口和托盘状态。
4. 查看控制台或日志错误。
5. 尝试退出应用。

预期结果：

- 应用不崩溃。
- 输出 JSON 解析失败信息。
- 托盘仍可操作。

通过标准：

- 错误信息明确包含解析失败原因或文件路径。
- 应用仍可退出。

### 6.3 pet.json 字段缺失

目标：

- 验证 `pet.json` 缺少关键字段时有明确兜底或错误提示。

测试字段：

- `spritesheet`。
- `frame.width`。
- `frame.height`。
- `frame.columns`。
- `frame.rows`。
- `animations.idle`。
- `animations.idle.frames`。
- `animations.idle.fps`。

步骤：

1. 分别移除一个关键字段。
2. 启动应用。
3. 观察应用行为。
4. 查看控制台或日志错误。

预期结果：

- 应用不崩溃。
- 输出字段缺失或 manifest 校验失败信息。
- 如果有默认值策略，应按默认值运行。
- 如果没有默认值策略，应进入可控错误状态。

通过标准：

- 每个关键字段缺失都有明确处理。
- 不出现卡死、无限重试或空白高 CPU 状态。

### 6.4 spritesheet.webp 缺失

目标：

- 验证缺少 spritesheet 图片时应用不会崩溃。

步骤：

1. 保留合法 `pet.json`。
2. 删除或重命名 `spritesheet.webp`。
3. 启动应用。
4. 观察应用行为。
5. 查看控制台或日志错误。
6. 尝试通过托盘退出应用。

预期结果：

- 应用不崩溃。
- 输出图片缺失错误。
- 托盘入口可用。
- 不持续高频重试加载图片。

通过标准：

- 错误信息能定位到 `spritesheet.webp` 缺失。
- 应用保持可退出。

### 6.5 spritesheet.webp 损坏

目标：

- 验证图片文件存在但无法解码时应用不会崩溃。

步骤：

1. 使用非图片内容替换 `spritesheet.webp`，或使用损坏图片。
2. 启动应用。
3. 观察应用行为。
4. 查看控制台或日志错误。
5. 观察 CPU 是否异常。

预期结果：

- 应用不崩溃。
- 输出图片加载或解码失败错误。
- 不进入无限加载或高频重试。

通过标准：

- 错误信息明确。
- 应用仍可退出。
- CPU 保持低占用。

### 6.6 动画帧索引越界

目标：

- 验证 `animations.idle.frames` 包含超出 spritesheet 范围的帧索引时不会崩溃。

步骤：

1. 在 `pet.json` 中将 idle frames 设置为包含越界值，例如 `[0, 1, 999]`。
2. 启动应用。
3. 观察动画播放。
4. 查看控制台或日志错误。

预期结果：

- 应用不崩溃。
- 输出帧索引越界错误。
- 可选择跳过非法帧或停止播放并进入错误状态。

通过标准：

- 不绘制错误内存区域或异常空白循环。
- 错误可定位到具体动画和帧索引。

### 6.7 fps 配置异常

目标：

- 验证 fps 为 0、负数、过高或非数字时不会造成异常渲染。

测试值：

- `0`。
- `-1`。
- `1000`。
- `"fast"`。
- `null`。

步骤：

1. 分别设置异常 fps。
2. 启动应用。
3. 观察动画和 CPU。
4. 查看错误或校验日志。

预期结果：

- 应用不崩溃。

## 9. 缺陷分级

### 9.1 P0 阻塞

- 应用无法启动。
- 应用启动即崩溃。
- 无法退出应用。
- 托盘和窗口均不可控。
- 隐藏后应用进入持续高 CPU 且无法恢复。

### 9.2 P1 严重

- 透明窗口显示异常，出现明显背景矩形。
- idle 动画无法播放。
- 资源异常导致卡死。
- 右键菜单或托盘核心控制不可用。
- 拖拽导致应用崩溃。

### 9.3 P2 一般

- 动画偶发跳帧。
- 拖拽不够流畅。
- 菜单状态文字不准确。
- 错误日志不够清晰。
- 隐藏 / 显示后位置不符合预期。

### 9.4 P3 优化

- CPU 或内存仍可进一步降低。
- 日志格式需要优化。
- 菜单交互细节需要调整。
- 测试辅助信息需要补充。
