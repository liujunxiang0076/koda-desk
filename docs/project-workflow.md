# 项目工作流规则

## 何时读取

当任务需要构建、测试、安装依赖、修改项目、提交代码、删除/移动文件，或涉及 Python、Node、包管理器、Git、安全边界时，先读取本文件。

## 项目命令

- 运行项目命令前先检查项目文件和已有 scripts。
- Node 包管理器按锁文件选择；如果存在多个锁文件，不擅自选择，先说明冲突并询问用户。
- 用户要求文档、配置或代码落地时，优先直接修改相关文件并验证，不停留在建议层。

## Git 与文件安全

- 如果工作区已有与当前任务无关的改动，不要回滚；只处理当前任务范围。
- 不执行 `git reset --hard`、`git clean -fdx` 等破坏性 Git 命令，除非用户明确要求。
- 删除或移动文件前必须确认目标路径；不要把未验证变量传给删除命令。
