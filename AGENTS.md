# AGENTS.md

本文件为在本仓库中工作的 AI coding agents 提供通用协作约定。除非用户另有明确要求，请优先遵循这些规则。

## 工作原则

- 先理解现有代码结构和约定，再进行修改。
- 保持变更范围最小，避免无关重构、格式化或文件移动。
- 不要覆盖、回滚或删除用户已有改动，除非用户明确要求。
- 对不确定的行为、接口或需求，优先从代码、测试和文档中验证。
- 修改完成后尽量运行相关测试或静态检查，并在回复中说明验证结果。

## 代码修改

- 遵循项目已有的语言、框架、目录结构和命名风格。
- 优先复用现有工具函数、组件、服务和配置。
- 只在确有必要时新增抽象；不要为了小改动引入复杂结构。
- 注释应解释不明显的原因或约束，避免重复描述代码本身。
- 不要提交生成物、缓存、日志或本地环境文件，除非项目明确需要。

## 测试与验证

- 对行为变更添加或更新对应测试。
- 对 bug 修复优先补充能复现问题的测试。
- 如果无法运行测试，应说明原因和潜在风险。
- 不要为了让测试通过而弱化断言或隐藏真实问题。

## Git 与文件安全

- 在操作前注意工作区可能已有用户改动。
- 不使用破坏性 Git 命令，例如 `git reset --hard` 或强制 checkout，除非用户明确要求。
- 不修改与当前任务无关的文件。
- 涉及凭据、密钥、令牌或个人数据时，不要写入仓库或输出到日志。

## 交付说明

- 最终回复应简洁说明完成了什么、修改了哪些关键文件、做过哪些验证。
- 若存在未完成项、假设或风险，应明确列出。
- 给出后续建议时，应与当前任务直接相关。
